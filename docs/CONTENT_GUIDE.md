# Content Guide

## Purpose

This document defines how content should be written, structured, and maintained for the Uptown Gaming website MVP.

Cursor should use this file to keep site content:
- clear
- brand-aligned
- mobile-friendly
- easy to scan
- conversion-aware
- consistent across pages

This document is about **content strategy and writing rules**, not final visual styling.

---

## Content Principles

### 1. Brand Clarity First
The content should quickly explain what Uptown Gaming is:
- a gaming and community brand
- an event-driven space
- a place for casual and competitive experiences
- a venue that can also be booked

Users should understand this fast without needing to read long paragraphs.

### 2. Event and Booking Focus
Content should support the primary site goals:
- communicate the brand and culture
- show venue and event value
- make it easy to book the space
- make it easy to ask questions or follow along

Do not let the site read like a restaurant website.

### 3. Mobile-First Writing
Most users will scan content, not read every word.
Write for:
- short attention spans
- small screens
- fast understanding
- clear next actions

### 4. Clear Before Clever
The content can have personality, but clarity comes first.
Avoid vague marketing copy that sounds polished but says little.

### 5. Structured and Reusable
Content should fit the approved content model and page specs.
Whenever possible, write content in a way that can be reused across:
- homepage previews
- section summaries
- route intros
- cards
- CTA blocks

### 6. Community-Centered Tone
The brand should feel:
- welcoming
- confident
- energetic
- polished
- local
- community-driven

---

## Brand Voice

### Voice Attributes
The website voice should feel:
- bold
- clear
- friendly
- trustworthy
- community-centered
- event-driven
- modern without sounding generic

### Tone Balance
The copy should feel energetic, but not chaotic.
It should feel polished, but not corporate.
It should feel accessible, but not flat.

### Avoid
- overly corporate language
- startup buzzwords
- vague hype language
- excessive slang
- trying too hard to sound edgy
- filler copy that does not help the user

---

## Messaging Priorities

Across the site, the content should reinforce these ideas:

### 1. What Uptown Gaming Is
A gaming and community brand centered on events, experiences, and local impact.

### 2. What People Can Do Here
- attend events
- learn about the venue
- book the space
- connect with the community
- reach out for more information

### 3. Why It Matters
Uptown Gaming is not just a name or venue listing.
It represents a community-centered event experience with real identity.

### 4. What the User Should Do Next
The content should always make the next step feel obvious:
- view events
- explore the venue
- book the space
- contact the team
- follow on social

## Audience-Aware Writing

### Audience 1 — Local Visitor / Community Member
This user wants to know:
- what Uptown Gaming is
- what kinds of events happen
- whether they should attend
- whether the vibe feels welcoming and real

#### Writing Guidance
- keep the value proposition immediate
- show event/community energy clearly
- use clear event-oriented language
- make event discovery easy

---

### Audience 2 — Organizer / Booker
This user wants to know:
- whether the venue is bookable
- what the space offers
- whether it fits their needs
- how to inquire

#### Writing Guidance
- be direct about booking
- highlight venue value and use cases
- avoid making them dig for next steps
- support trust with clear, useful details

---

### Audience 3 — Curious Social Visitor
This user may land from:
- Instagram
- event promotions
- word of mouth
- a direct shared link

They need fast answers.

#### Writing Guidance
- explain the brand quickly
- use strong headings and short support copy
- make the site understandable without deep scrolling
- keep primary CTAs visible

---

## Content Hierarchy Rules

Every page and section should support a simple hierarchy:

### 1. Primary Message
What this page or section is about.

### 2. Supporting Message
Why it matters.

### 3. Action
What the user should do next.

This should appear in:
- hero sections
- page intros
- venue sections
- booking sections
- CTA areas

### Good Pattern
- heading
- short supporting paragraph
- CTA or structured content block

### Avoid
- long walls of text before any useful action
- burying the key point in the third paragraph
- mixing too many messages in one section

---

## Writing for Mobile

### Mobile Content Rules
- prefer short paragraphs
- keep headings specific
- break content into clear sections
- use bullets only when they improve scanning
- avoid dense intro copy
- keep CTA language short and obvious

### Paragraph Length
Prefer:
- 1 to 3 short sentences per paragraph
- concise section intros
- short card descriptions

Avoid:
- oversized paragraphs
- stacked marketing copy with no structure
- repeating the same idea in multiple ways

---

## Heading Rules

Headings should do real work.

### Good Headings Should
- explain the section clearly
- help scanning
- support SEO intent where appropriate
- feel on-brand without becoming vague

### Good Examples
- Upcoming Events
- Book the Space
- A Venue Built for Community Events
- What Uptown Gaming Is About
- Host Your Next Event Here

### Avoid Headings Like
- Welcome
- Discover More
- Experience the Difference
- Elevate Your Vision
- The Ultimate Destination

These are too vague unless the supporting context is extremely strong.

---

## CTA Copy Rules

CTA language should be action-oriented and easy to understand.

### Preferred Primary CTAs
- Book the Space
- Start Booking Inquiry
- Inquire About the Space

### Preferred Secondary CTAs
- View Events
- Explore the Venue
- Contact Us
- Learn More
- Follow on Instagram

### CTA Rules
- keep button labels short
- do not use overly clever CTA wording
- keep CTA wording consistent across pages
- do not use multiple synonyms for the same action unless there is a reason

### Avoid
- Let’s Go
- Join the Movement
- Unlock the Experience
- Start Your Journey

These may sound dramatic but reduce clarity.

## Route-Level Content Guidance

## `/` — Home

### Home Content Goal
Give users a strong first impression and quickly answer:
- what Uptown Gaming is
- what happens here
- why it matters
- what they should do next

### Home Content Priorities
1. clear brand identity
2. event/community value
3. venue/space value
4. booking CTA
5. supporting trust and social proof

### Home Content Rules
- keep hero copy short and strong
- use sections to summarize, not overload
- preview events instead of dumping too much detail
- include at least one strong booking path
- make the site feel alive and active

### Recommended Home Content Blocks
- Hero headline
- Hero support copy
- Primary CTA
- Secondary CTA
- Brand intro
- Events preview
- Venue highlight
- Booking CTA
- Optional community/support block

---

## `/events` — Events

### Events Page Goal
Help users understand the types of events Uptown Gaming runs or hosts.

### Events Content Priorities
1. event clarity
2. event variety
3. event energy
4. next-step action

### Events Page Rules
- use clear event titles
- include date/time/status when available
- support both discovery and excitement
- keep event descriptions useful, not overloaded
- include a next-step CTA

### Event Preview Copy Should Include
- title
- date
- time
- short description
- CTA if relevant

### Event Description Rules
- short descriptions should be concise and useful
- avoid writing every event the same way
- do not rely only on hype language
- make sure a user can tell what the event actually is

---

## `/book` — Book the Space

### Book Page Goal
Turn interested organizers and collaborators into inquiries.

### Book Page Content Priorities
1. clear booking invitation
2. who the space is for
3. why the venue is valuable
4. how inquiry works
5. FAQ or expectation-setting

### Book Page Rules
- make the page feel easy and approachable
- do not sound bureaucratic
- clearly explain the inquiry path
- focus on fit, value, and action
- avoid overpromising details that are not confirmed

### Strong Book Page Topics
- host tournaments
- community gatherings
- special events
- branded experiences
- custom activations if approved

### Book Page Copy Should Answer
- who should reach out
- what the space may be good for
- what makes the venue useful
- how to start the process

---

## `/venue` — Venue

### Venue Page Goal
Explain the physical and experiential value of the space.

### Venue Page Content Priorities
1. what the venue offers
2. what it is good for
3. what features matter
4. why it supports events well
5. booking CTA

### Venue Page Rules
- focus on practical value and atmosphere
- keep feature descriptions concise
- organize amenities clearly
- support both emotional and functional understanding

### Venue Feature Copy Should
- use short, scannable descriptions
- explain why a feature matters
- avoid generic filler like “top-tier” unless supported by specifics

---

## `/about` — About

### About Page Goal
Explain who Uptown Gaming is, why it exists, and what it stands for.

### About Content Priorities
1. mission
2. identity
3. community relevance
4. trust and authenticity
5. supporting CTA

### About Page Rules
- tell a clear story without becoming overly long
- keep mission and values understandable
- highlight community roots where helpful
- do not make the page feel generic

### About Content Should Reinforce
- local identity
- community-building
- event-centered purpose
- credibility and care

---

## `/contact` — Contact

### Contact Page Goal
Make it easy for users to reach out.

### Contact Content Priorities
1. clarity
2. low friction
3. obvious next step
4. confidence that someone will respond

### Contact Page Rules
- keep the intro short
- explain what the user can contact about
- make form labels direct
- avoid long paragraphs

### Good Contact Intro Topics
- event questions
- booking inquiries
- partnership questions
- general outreach

## Section Content Rules

## Hero Section
The hero should communicate:
- what Uptown Gaming is
- what kind of experience it offers
- what action the user should take

### Hero Rules
- headline should be concise
- support copy should be short
- CTA should appear early
- do not overload the hero with too much text
- hero should still work if users only read the first screen

---

## Brand Intro Section
This section should explain the brand more clearly.

### Brand Intro Rules
- explain the value proposition
- reinforce community and event identity
- stay concise
- avoid repeating the hero word-for-word

---

## Events Preview Section
This section should show momentum and relevance.

### Events Preview Rules
- use short event previews
- keep scanning easy
- make the “view more” path obvious
- do not overcrowd the homepage with too many event items

---

## Venue Features Section
This section should show why the space matters.

### Venue Features Rules
- feature titles should be clear
- descriptions should be short
- connect features to usefulness
- focus on event and venue value, not random amenities

---

## Booking CTA Section
This section should convert interest into inquiry.

### Booking CTA Rules
- use direct language
- make the action obvious
- support the CTA with a short reason to act
- do not add too much explanation around the CTA

---

## Card Copy Rules

Cards should be short and highly scannable.

### Event Cards
Should usually include:
- title
- date/time or status
- short description
- optional CTA

### Venue Feature Cards
Should usually include:
- feature name
- short benefit-focused description

### Partner or Community Cards
Should usually include:
- name
- short supporting description
- optional link

### Card Copy Rules
- do not overload cards with paragraphs
- keep titles specific
- keep descriptions short
- maintain consistent length where possible

---

## Form Copy Rules

Forms should feel easy to use.

### Form Intro Copy
Should explain:
- what the form is for
- what kinds of inquiries are welcome
- what happens next if appropriate

### Label Rules
- labels should be explicit
- do not rely only on placeholder text
- keep wording simple

### Helper Text Rules
- only include helper text when it adds real clarity
- keep error messages direct and human
- avoid technical language

### Good Field Labels
- Name
- Email
- Event Type
- Preferred Date
- Tell Us About Your Event

### Avoid
- vague field names
- internal language
- labels that assume too much context

---

## SEO-Aware Content Rules

The content should support distinct page intent without keyword stuffing.

### Route Intent Reminder
- Home = brand overview + primary conversion
- Events = event discovery
- Book = booking inquiry
- Venue = amenities and space value
- About = mission and identity
- Contact = direct outreach

### SEO Writing Rules
- headings should reflect actual page purpose
- intro paragraphs should support page intent
- avoid duplicate intros across multiple routes
- do not repeat the same phrase unnaturally
- write for humans first

---

## Reusable Copy System

To make content easier to manage, write with reusable content layers in mind.

### Reusable Content Types
- short page intro
- section heading
- section support copy
- CTA label
- card description
- FAQ answer
- feature description

### Reuse Rules
- keep reusable copy modular
- avoid writing route-specific copy when the same message can be shared
- do not force reuse where a page needs more direct specificity

---

## Content Source Guidance

The build doc recommends starting with structured local content before adding a more complex content system.

### MVP Content Approach
Use:
- local structured content
- typed content objects
- reusable page and section data
- simple editable files

Do not add:
- a CMS too early
- a backend content system too early
- unnecessary content tooling in MVP

---

## Photography and Media Copy Support

The site should include some form of media and should feel active and real.

### Media-Adjacent Copy Should Do
- support the image, not narrate it awkwardly
- add context and value
- help users understand what they are seeing

### Avoid
- captions that say nothing meaningful
- repeating the exact headline next to every image
- using media with no content purpose

---

## Content QA Checklist

Before approving content, confirm:

- it clearly explains what Uptown Gaming is
- it supports event and booking goals
- it feels community-centered
- it stays mobile-friendly and scannable
- headings are clear
- CTA copy is consistent
- route copy matches route purpose
- content does not drift into restaurant-first framing
- copy feels energetic but controlled
- there is no empty marketing filler

---

## Final Rule

When writing or reviewing content, ask:

1. Does this clearly help the user understand Uptown Gaming?
2. Does this support events, venue value, booking, or contact?
3. Is this easy to scan on mobile?
4. Is this specific enough to be useful?
5. Does this sound like the brand without becoming vague or overhyped?

If not, simplify and rewrite.
